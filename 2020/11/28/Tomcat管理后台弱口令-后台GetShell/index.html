Tomcat管理后台弱口令+后台GetShell

环境搭建

使用 vulhub 靶场搭建，基于 docker 构建

下载项目

# 下载项目
wget https://github.com/vulhub/vulhub/archive/master.zip -O vulhub-master.zip
unzip vulhub-master.zip
cd vulhub-master

进入到 Tomcat 环境所在目录

cd tomcat/tomcat8

启动环境

docker-compose up -d

执行上述命令后，会自动下载所需镜像，创建容器

使用 docker ps 命令查看启动的容器

CONTAINER ID        IMAGE               COMMAND             CREATED              STATUS              PORTS                    NAMES
5a72857fd901        vulhub/tomcat:8.0   "catalina.sh run"   About a minute ago   Up About a minute   0.0.0.0:8080->8080/tcp   tomcat8_tomcat_1

容器进行了端口映射，将容器的8080端口映射到宿主机的8080端口

浏览器访问 http://192.168.182.129:8080 即可访问 Tomcat 环境

漏洞原理

在配置Tomcat的时候使用了常用的用户名和弱口令密码，可导致攻击者使用弱口令登录管理后台。

在 conf/tomcat-users.xml 文件中配置用户的权限：

<?xml version="1.0" encoding="UTF-8"?>
<tomcat-users xmlns="http://tomcat.apache.org/xml"
              xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
              xsi:schemaLocation="http://tomcat.apache.org/xml tomcat-users.xsd"
              version="1.0">

    <role rolename="manager-gui"/>
    <role rolename="manager-script"/>
    <role rolename="manager-jmx"/>
    <role rolename="manager-status"/>
    <role rolename="admin-gui"/>
    <role rolename="admin-script"/>
    <user username="tomcat" password="tomcat" roles="manager-gui,manager-script,manager-jmx,manager-status,admin-gui,admin-script" />
</tomcat-users>

上述配置用户tomcat拥有所有权限，密码是tomcat

注：正常安装的情况下，tomcat8中默认没有任何用户，且manager页面只允许本地IP访问。只有管理员手工修改了这些属性的情况下，才可以进行攻击。

在管理后台提供了对Web应用程序动态部署的功能，管理员仅需要上传一个war格式的文件就能部署web应用程序，非常方便。攻击者在成功登录管理后台后，可以利用war包部署功能上传恶意脚本获取网站控制权

漏洞复现

弱口令登录管理后台

访问Tomcat默认管理后台地址 http://192.168.182.129:8080/manager/html，弹框提示要输入账号和密码

使用弱口令 tomcat/tomcat，成功登录管理后台

常见弱口令

admin:admin
admin:123456
admin:tomcat
tomcat:tomcat
tomcat:admin
tomcat:123456t

使用BurpSuite来爆破

抓取登录认证请求数据包，发送到Intruder

设置 Payload type为 custom iterator（自定义迭代器）

设置Payload

Position 1 添加用户字典
Position 2 添加 : 字符
Position 3 添加密码字典

注：Tomcat管理后台使用BASIC认证，在HTTP请求头中有一个Authorization字段，其值为 username:password 的Base64编码

Payload Processing选择 Encode base64

Payload Encoding去掉勾选 URL-encode

开始爆破，爆破完成后，查看有无状态码为200的请求，有表示爆破成功

注：请求响应状态码401（未经授权：访问由于凭据无效被拒绝），响应状态码200表示登录成功

管理后台手动部署WAR包GetShell

首先登录管理后台，滑动页面到 war file to deploy 处

上传包含木马的WAR包GetShell

WAR包，文件后缀名为 .war，其本质上是一个压缩包格式的文件。将冰蝎的jsp马打包成压缩包文件，然后将后缀名改为 .war

上传该文件，点击Deploy按钮进行上传部署，成功后可以在 applications 栏中看到部署的应用

对应访问地址是：http://192.168.182.129:8080/shell/shell.jsp

使用冰蝎连接

修复建议

  1. 如果不需要重新部署或部署新的Web应用，可以将管理后台删除

  2. 为管理后台设置复杂的用户密码
