BurpSuite扩展插件

BurpSuite提供了支持第三方拓展插件的功能，方便使用者编写自定义插件或从插件商店中安装扩展插件

运行BurpSuite，点击Extender标签，进入Extender面板，会看到四个子面板

  • Extensions（管理和加载burp插件）
  • BApp Store（插件应用商店）
  • APls（接口文档）
  • Options（其他选项设置）

Extensions子面板

在这选项面板会显示已安装的插件列表，通过对插件列表的管理，我们可以对插件进行后期的维护，这里也可以加载插件。

加载插件

插件配置

如果我们想对某个插件的配置信息进行编辑，可以选中某个插件，则其下方的Details和Output标签页会显示插件信息。

BApp Store子面板

Burp应用商店，在这里提供各种Burp的插件安装。在这里我们可以安装自己需要的插件，也可以安装应用商店没有的插件。

当一个插件安装完毕后，在Burp Extender面板的Extension选项界面中，会自动显示已加载的插件列表。

APIs子面板

该选项展示了编写自定义插件BurpSuite所提供使用的接口类的定义和描述。

在该选项面板最下面有两个按钮，Save interface files（保存为接口文件）和Save Javadoc files（保存为Javadoc文件）

当点击 Save interface files 按钮并指定存储目录后，会在指定目录下生成一系列Java文件，这些文件的内容为接口类的定义和描述。

当点击Save Javadoc files按钮并指定存储目录后，会在存储目录生成与API对应的Javadoc文件，使用浏览器打开

Options子面板

该选项主要用于配置使用不同语言编写的插件需要的运行环境，BurpSuite支持的编写插件的语言有Java、Python、Ruby，而运行使用Python或Ruby编写的插件需要配置兼容Java与Python、Java与Ruby的Jar，默认情况下，BurpSuite支持的为Jython和Jruby。

Jython下载地址

https://www.jython.org/download.html

FastjsonScan

一个简单的Fastjson反序列化检测burp插件

使用方法也很简单，就像使用repeater一样，你可以在burp的任何地方选中一个请求右键选择 Send to FastjsonScan 将这个请求发送到Fastjson Scan，然后就只需要等待扫描结束。如果扫描的目标存在漏洞，在窗口下面的Request窗口会展示使用的payload，如果没有漏洞，则会展示原始的请求与响应

ps: 由于反序列化检测利用了dnslog，所以检测会稍微慢一点，在等待结果期间你还可以继续看其他请求，真的是相当方便呢

配置本地仓库

如果不配置本地仓库路径，默认会在 C:\Users\<user>\.m2\repository\ 目录下存放从远程下载到的包

打开maven安装目录下的 conf 目录下的 setting.xml 文件，配置指定本地仓库路径

安装 jsEncrypter

项目地址

https://github.com/c0ny1/jsEncrypter

将安装包下载下来后进行编译

在 jsEncrypter 项目目录下，打开cmd窗口，输入 mvn package 命令编译插件，编译成功后会生成一个target目录

将 jar 插件添加到 burpsuite

使用

将前端用于加密的 js 代码编写到 jsEncrypter 项目目录下的script目录下的 phantomjs_server.js 文件中

// 处理函数
function js_encrypt(payload){
    var newpayload;
    /**********在这里编写调用加密函数进行加密的代码************/

    /**********************************************************/
    return newpayload;
}

例如前端使用 sha1 加密，将代码改写为，并将文件另存为 jsEncrypter_sha1.js

// 处理函数
function js_encrypt(payload){
    var newpayload;
    /**********在这里编写调用加密函数进行加密的代码************/
    newpayload = hex_sha1(payload);
    /**********************************************************/
    return newpayload;
}

使用 phantomjs 运行这个 js 文件

phantomjs jsEncrypter_sha1.js

如果没有错误，会显示successful

然后在burpsuite打开 jsEncrypter 插件，默认 Host:127.0.0.1 port:1664，点击connect按钮进行连接，连接成功会显示true，然后点击test，它会根据你的算法帮你在右边生成密文

结合intruder使用

抓取数据包放入intruder模块，在payload面板的Payload Processing处

点击Add按钮，添加对payload的处理规则，选择Invoke Burp extension，select processor: jsEncrypter

点击start attack按钮开始爆破，会自动使用 jsEncrypter 插件对payload进行加密处理

爆破成功后，在 phantomjs 输出信息中查找明文即可

HaE(信息高亮标记与提取)

HaE(Highlighter and Extractor) 是基于 BurpSuite 插件 JavaAPI 开发的请求高亮标记与信息提取的辅助型插件。

该插件可以通过自定义正则的方式匹配响应报文或请求正文，可以自行决定符合该自定义正则匹配的相应请求是否需要高亮标记、信息提取。

注：HaE的使用，对测试人员来说需要基本的正则表达式基础，由于 Java 正则表达式的库并没有 Python 的优雅或方便，在使用正则的，HaE要求使用者必须使用 () 将所需提取的表达式内容包含；例如你要匹配一个Shiro应用的响应报文，正常匹配规则为 rememberMe=delete，如果你要提取这段内容的话就需要变成 (rememberMe=delete)

项目地址：

https://github.com/gh0stkey/HaE

安装：

下载 releases 版本，插件装载: Extender - Extensions - Add - Select File - Next

初次装载HaE会初始化配置文件（config.json），默认配置文件内置一个正则: Email，初始化的配置文件会放在与BurpSuite Jar包同级目录下。

除了初始化的配置文件外，还有init.hae文件，该文件用于存储配置文件路径；HaE支持自定义配置文件路径，你可以通过点击Select File按钮进行选择自定义配置文件。

Burp Suite扩展可从JSON响应中查看和提取数据。这插件方便你从大量json返回包中提取你所需要返回的敏感信息字段的值。

在go的按键左边的填框里填写你需要提取的json字段的名即可匹配提取了。

项目地址

https://github.com/augustd/burp-suite-jsonpath

可以将请求复制到剪贴板，作为多种编程语言的函数。

Xss-Sql-Fuzz

用来fuzz xss和sql注入, 可以对post，get 的所有参数一键自动添加上payload

JSON Beautifier(格式化查看 json)

JSON Beautifier 可以使 json 格式的数据更加好看，也就是格式化查看，在查看 json 请求时十分好用。该插件可以直接在Bapp Store 安装。

注：在2020.7及以后的版本的BurpSuite自带Pretty功能可以实现美化JSON

Software Vulnerability Scanner(自动根据版本号查找 CVE)

Software Vulnerability Scanner 是一个扫描器增强插件，它会检查网站的一些软件版本信息，然后通过 vulners.com
上的漏洞数据库来查询相应的 CVE 编号，找到的结果会显示在漏洞面板上，不用我们自己手动去查找某个版本的 CVE 。该插件可以直接在Bapp Store 安装。

Autorize(强大的越权自动化测试工具)

Autorize 可以用来测试权限问题，可以在插件中设置一个低权限账号的 cookie ，然后使用高权限的账号去浏览所有功能，Autorize 会自动用低权限账号的 cookie 重放请求，同时也会发一个不带 cookie 的请求来测试是否可以在未登录状态下访问。该插件可以直接在Bapp Store 安装，插件使用Python编写，运行需要配置Jython环境。

Logger++(更强大的请求历史查看器)

Logger++ 可以查看所有工具发出的请求，如 repeater、intruder、scanner、插件等。这样我们可以查看到扫描时的一些 payload，以及在扫描时监控网站后台的响应情况。该插件可以直接在Bapp Store 安装。

Brida(连接 frida 与 burpsuite)

Brida 可以连接 frida 与 burpsuite ，hook 神器 frida 在移动 APP 的渗透上发挥着很重要的作用，如绕过 ssl pinning ， 自动加解密 APP 的请求内容等。我们可以把一些常用的 frida 脚本放在 Brida 的脚本上，然后通过 Brida 启动 APP来进行渗透

sqlmap4burp++(连接 burpsuite 与 sqlmap)

sqlmap4burp++是一款兼容Windows、MacOS，Linux多个系统平台的BurpSuite与SQLMap联动插件。可以方便地对请求调用 SQLMap 来扫描。

项目地址：

https://github.com/c0ny1/sqlmap4burp-plus-plus
CSRF Token Tracker(绕过CSRF限制进行暴力破解)

CSRF Token Tracker 可以自动获取 csrf 的 token，对于一些有 csrf 限制的请求，它可以绕过该限制，如暴力破解具有 csrf token 的登录请求。该插件可以直接在Bapp Store 安装。

Decompressor(自动解码和修改 gzip 压缩包)

Decompressor 可以自动解码和修改使用 gzip 压缩过的请求数据。有时候网站发送的数据是使用 gzip 压缩过的，使我们在 burpsuite 上看到的请求是乱码，并且无法修改请求数据，Decompressor可以帮我们自动解码来查看，并且可以以未压缩的方式修改数据，然后再自动进行压缩。该插件可以直接在Bapp Store 安装。

Wsdler(测试 WSDL 请求)

Wsdler 可以解析 WSDL 请求，以便使用 repeater 和 scanner 对 WSDL 请求进行测试。该插件可以直接在Bapp Store 安装。
